User Management

# Method Endpoint Description
1 POST api/auth/users/employee/all Paginated, filtered & sorted list of all employees
2 GET api/auth/users/role Fetch all available roles — used for role assignment dropdown
3 GET api/auth/users/commercial/roles Fetch all available commercial roles
4 POST api/auth/users/employee/exists Verify if an employee email already exists before adding
5 POST api/auth/users/employee Create or update an employee — role assignment and role updates use the same API
6 GET api/auth/users/employee/{userId} Fetch full profile of a specific employee by ID
7 PUT api/auth/users/employee/status Enable or disable a user account
8 POST api/auth/users/employee/csv Export the filtered user list as a CSV file

Role Assignment Flow

Step-by-step sequence for assigning a role to a user

1
Fetch all available roles for the dropdown or only commercial roles
GET  api/auth/users/role or /api/auth/users/commercial/roles
2
Verify the employee email doesn't already exist
POST  api/auth/users/employee/exists
3
Create user with the selected role assigned
POST  api/auth/users/employee
4
Optionally — fetch existing employee profile
GET  api/auth/users/employee/{userId}
5
Enable or disable the user account
PUT  api/auth/users/employee/status

Insert/Update Role Script

SQL script to insert or update commercial leasing roles, then verify the data.

SQL Script
-- UPSERT using MERGE
INSERT INTO wasl_identity.Roles (Id, Name, Code, CreatedAt, UpdatedAt)
VALUES
    (20, 'CL Admin', 'CL_ADMIN', NOW(), NOW()),
    (21, 'CL Manager', 'CL_MANAGER', NOW(), NOW()),
    (22, 'CL Agent', 'CL_AGENT', NOW(), NOW()),
    (23, 'CL Committee', 'CL_COMMITTEE', NOW(), NOW())
ON DUPLICATE KEY UPDATE
    Name = VALUES(Name),
    Code = VALUES(Code),
    UpdatedAt = NOW();

-- Check existing data
SELECT * FROM wasl_identity.Roles;
